# CurrencySystem

## Описание проекта

**CurrencySystem** — клиент-серверное приложение для получения актуальных курсов валют из публичных API, хранения истории котировок, построения диаграмм и экспорта данных в файлы. Рассчитано на параллельную работу нескольких пользователей.

---

## Функциональные требования

### Клиент

1. **Подключение**
   - Пользователь задаёт адрес и порт сервера.
   - Клиент отображает статус соединения.
   - Клиент показывает понятные ошибки: таймаут / сервер недоступен / неверный адрес.

2. **Выбор источника котировок**
   - Клиент предоставляет выбор источника котировок.
   - Если выбранный API недоступен — клиент показывает ошибку и даёт переключиться на другой источник.

3. **Актуальные курсы**
   - Клиент отображает список валют: код / название.
   - Клиент отображает текущие значения:
     - для выбранной валюты/пары или
     - относительно базовой валюты.
   - Клиент отображает время актуальности котировок.

4. **История котировок**
   - Клиент позволяет выбрать:
     - валюту/пару,
     - период from -> to,
     - шаг (например 1h / 5d / 1m / 1y).
   - Клиент отображает историю в виде таблицы.

5. **Диаграммы**
   - Клиент строит график по историческим данным.
   - Клиент позволяет просматривать значение точки на графике: время + курс.

6. **Конвертация**
   - Клиент принимает входные данные: сумма, валюта из, валюта в.
   - Клиент отображает: результат, курс, время, по которому выполнён расчёт.

7. **Экспорт**
   - Клиент сохраняет данные (актуальные и/или история) в форматы: json / csv / xlsx.
   - Клиент сохраняет график в файл (например, png). 

---

### Сервер

1. **Сетевое взаимодействие**
   - Сервер принимает подключения клиентов.
   - Сервер обеспечивает параллельную работу с несколькими клиентами.
   - Сервер обрабатывает запросы с таймаутами.

2. **Справочник валют**
   - Сервер хранит список поддерживаемых валют.
   - Сервер выдаёт справочник валют клиенту по запросу.

3. **Источники данных**
   - Сервер поддерживает несколько провайдеров котировок (например, ЦБ РФ и т.п.).
   - Сервер приводит данные к единому формату на выходе.

4. **Актуальные курсы**
   - Сервер получает и обновляет котировки.
   - Сервер хранит актуальные значения.
   - Сервер выдаёт актуальные курсы по запросу клиента.

5. **История котировок**
   - Сервер сохраняет котировки с временным шагом.
   - Сервер выдаёт историю за период.
   - Сервер поддерживает шаг/агрегацию, чтобы не отдавать слишком много точек.

6. **Конвертация**
   - Сервер выполняет конвертацию по последнему курсу.
   - Сервер возвращает: результат, использованный курс, временной шаг/время, по которым выполнён расчёт.

7. **Ошибки и валидация**
   - Сервер валидирует параметры запроса: валюта / период / шаг.
   - Сервер возвращает корректные ошибки: неверные аргументы / нет данных / API недоступен.

8. **Устойчивость**
   - При недоступности внешнего API сервер не падает.
   - Сервер отдаёт последние сохранённые данные и логирует проблему.
  
9. **Кеширование**
   - Сервер использует in-memory кеш для ответа на повторяющиеся запросы, чтобы уменьшить число обращений к внешним API и ускорить ответы.
   - Кеш применяется к:
     - актуальным курсам,
     - истории котировок.
   - Для каждой записи кеша задаётся TTL (время жизни). По истечении TTL данные считаются устаревшими и должны быть обновлены.
   - При недоступности внешнего API сервер может вернуть последнюю успешно закешированную версию данных (даже если TTL истёк) и записать событие в лог.
